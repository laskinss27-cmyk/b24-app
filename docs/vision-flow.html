НАШ СКЛАД

Хозяин этих данных: Остатки по складамПриходПеремещениеСписаниеРеализацияСебестоимость
Тут живёт точность:
одна цифра остатка, транзакции, без рассинхрона. Это то, чего Битрикс не умеет.
Почему нет «ада синхронизации»: остаток — это ОДНА цифра, и живёт она только в складе. Битрикс её не копирует, а показывает живьём по запросу. Туда-сюда летают события («спиши», «оприходуй»), а не два разных остатка, которые надо сверять. У каждого типа данных — один хозяин.
